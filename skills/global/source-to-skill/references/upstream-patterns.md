# Upstream Patterns

Use these as design patterns, not as templates to copy blindly.

## Local Patterns

- `web-page-capture`
  - good for turning URLs into reusable local artifacts before distillation
- `research-paper-reading-compass`
  - good for structured extraction from theory-heavy or academic material
- `codex-session-evolution`
  - good example of turning raw evidence into sanitized packets instead of copying full sources
- `skill-creator`
  - the correct downstream owner once the packet is stable enough to create or update a skill

## External Patterns

- `colleague-skill`
  - strong signal for person-to-skill distillation and expert knowledge capture
- `nuwa-skill`
  - strong signal for multi-source research, synthesis, generation, and validation loops
- `darwin-skill`
  - useful reminder that the resulting skill should later be evaluated and iterated
- `gbrain`
  - useful reminder to separate durable truth from volatile runtime behavior
- `claude-skillkit`
  - useful signal that documents can be converted into skills, but still need a bounded packet and overlap check

## Pattern Rule

Borrow:
- extraction contracts
- source separation
- validation loops
- distinct ownership

Do not borrow blindly:
- repository structure
- scoring systems
- large source dumps into `SKILL.md`
- marketing language about autonomous creation
